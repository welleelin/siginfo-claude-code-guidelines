🔄 三种工作流

根据任务复杂度自动选择

⚡
Quick Flow
复杂度 1-3 | 30分钟-2小时
5 步骤：输入→规划→开发→验证→检查
📋
Standard Flow
复杂度 4-6 | 2-8小时
8 阶段：准备→规划→质量→TDD→API→E2E→安全→最终
🏢
Enterprise Flow
复杂度 7-10 | 1-4周
5 阶段：分析→规划→方案→实现→质量保障

⚡ Quick Flow 详情

1
任务输入 - 快速评估任务复杂度
2
快速规划 - 生成简洁实现计划
3
快速开发 - 测试→实现→审查，覆盖率 85%+
4
验证测试 - 运行测试，100% 通过
5
质量检查 - 代码审查、覆盖率检查

📋 Standard Flow (8 阶段)

Phase 1-2
会话启动准备 → 任务规划 需确认
Phase 3-4
代码质量门禁 → TDD 开发 (RED→GREEN→REFACTOR)
Phase 5-6
API 完整性门禁 → E2E 测试
Phase 7-8
安全性门禁 → 最终质量门禁

# 1. 克隆项目

cd siginfo-claude-code-guidelines

# 2. 安装
./scripts/install.sh

# 3. 初始化记忆
./scripts/init-memory.sh

# 4. 开始任务
/plan "实现用户登录功能"
