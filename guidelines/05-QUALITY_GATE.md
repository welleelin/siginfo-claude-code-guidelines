## 🔧 自动化检查

### 代码质量检查

```bash
# 检查代码规范
npm run lint

# 检查类型
npm run type-check

# 代码审查
/code-review
```

### 构建检查

```bash
# 检查构建
npm run build

# 检查类型
npm run type-check
```

### 测试检查

```bash
# 运行所有测试
npm test

# 检查覆盖率
npm test -- --coverage

# 运行 E2E 测试
npx playwright test
```

```bash

./scripts/check-api-completeness.sh api-checklist.md
```


## 📊 质量指标

| 指标 | 最低要求 | 推荐值 | 说明 |
|------|---------|--------|------|
| 单元测试覆盖率 | 80% | 90%+ | 代码覆盖率 |
| 集成测试通过率 | 100% | 100% | API 接口测试 |
| 联调测试通过率 | 100% | 100% | 前后端集成测试 |
| E2E 测试通过率 | 100% | 100% | 端到端测试 |
| 构建时间 | <5 分钟 | <2 分钟 | 编译速度 |
| 代码审查 | 无 CRITICAL/HIGH | 无问题 | 代码质量 |
| **生产环境质量** | **95% 无 Bug** | **98%+ 无 Bug** | **最终目标** |

### 生产环境质量目标

**95% 无 Bug 的定义**：

1. **Bug 率计算**：
   ```
   Bug 率 = (生产环境 Bug 数 / 总功能点数) × 100%
   无 Bug 率 = 100% - Bug 率
   ```

2. **Bug 分级**：
   | 级别 | 定义 | 影响 | 目标 |
   |------|------|------|------|
   | P0 - 致命 | 系统崩溃、数据丢失 | 无法使用 | 0 个 |
   | P1 - 严重 | 核心功能不可用 | 严重影响 | ≤1 个 |
   | P2 - 一般 | 功能异常但有替代方案 | 部分影响 | ≤3 个 |
   | P3 - 轻微 | UI 问题、体验问题 | 轻微影响 | ≤5 个 |

3. **质量保障措施**：
   - ✅ 前端单元测试覆盖率 ≥80%
   - ✅ 后端单元测试覆盖率 ≥80%
   - ✅ 集成测试通过率 100%
   - ✅ 联调测试通过率 100%（禁止 Mock）
   - ✅ E2E 测试通过率 100%（禁止 Mock）
   - ✅ 代码审查无 CRITICAL/HIGH 问题
   - ✅ 安全扫描无漏洞
   - ✅ 性能测试达标

4. **测试覆盖要求**：
   - ✅ 正常流程（Happy Path）
   - ✅ 异常流程（Error Handling）
   - ✅ 边界条件（Boundary Cases）
   - ✅ 并发场景（Concurrency）
   - ✅ 数据一致性（Data Consistency）

## 8. 确定性验证

### 验证内容

- **测试可重复性**：
  - ✅ 连续运行 3 次，结果完全一致
  - ✅ 不同时间运行，结果一致
  - ✅ 不同环境运行，结果一致

- **Mock 接口标记**：
  - ✅ 所有 Mock 接口都有 `// ⚠️ MOCK:` 标记
  - ✅ 标记包含 Mock 原因和预计替换时间
  - ✅ Mock 接口清单已记录到 MEMORY.md

- **不确定性隔离**：
  - ✅ 时间依赖已隔离（使用 `jest.useFakeTimers()`）
  - ✅ 随机性已隔离（使用固定种子）
  - ✅ 网络请求已 Mock（使用 MSW/nock）
  - ✅ 文件系统操作已确定性排序
  - ✅ 并发操作已确定性排序
  - ✅ 环境变量已固定配置

### 验证命令

```bash
# 快速检查（不运行测试）
./scripts/verify-determinism.sh --quick

# 完整验证（包含测试可重复性）
./scripts/verify-determinism.sh "测试套件名" 3

# 扫描 Mock 接口
./scripts/scan-mock-interfaces.sh
```

### 验证标准

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 测试可重复性 | 100% | 连续 3 次运行结果一致 |
| Mock 接口标记率 | 100% | 所有 Mock 接口都有标记 |
| 不确定性隔离率 | 100% | 所有不确定性来源已隔离 |
