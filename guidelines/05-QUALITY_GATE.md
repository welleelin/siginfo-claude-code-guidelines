## 🔧 自动化检查

### 构建检查

```bash
# 检查构建
npm run build

# 检查类型
npm run type-check

# 检查 Lint
npm run lint
```

### 测试检查

```bash
# 运行所有测试
npm test

# 检查覆盖率
npm test -- --coverage

# 运行 E2E 测试
npx playwright test
```

### 代码审查

```bash
# 自动代码审查
/code-review

# 安全审查
/security-review
```

### 确定性验证

```bash
# 快速检查（不运行测试）
./scripts/verify-determinism.sh --quick

# 完整验证（包含测试可重复性）
./scripts/verify-determinism.sh "用户登录" 3

# 扫描 Mock 接口
./scripts/scan-mock-interfaces.sh
```

## 📊 质量指标

| 指标 | 最低要求 | 推荐值 | 说明 |
|------|---------|--------|------|
| 单元测试覆盖率 | 80% | 90%+ | 代码覆盖率 |
| 集成测试通过率 | 100% | 100% | API 接口测试 |
| 联调测试通过率 | 100% | 100% | 前后端集成测试 |
| E2E 测试通过率 | 100% | 100% | 端到端测试 |
| 构建时间 | <5 分钟 | <2 分钟 | 编译速度 |
| 代码审查 | 无 CRITICAL/HIGH | 无问题 | 代码质量 |
| **生产环境质量** | **95% 无 Bug** | **98%+ 无 Bug** | **最终目标** |

### 生产环境质量目标

**95% 无 Bug 的定义**：

1. **Bug 率计算**：
   ```
   Bug 率 = (生产环境 Bug 数 / 总功能点数) × 100%
   无 Bug 率 = 100% - Bug 率
   ```

2. **Bug 分级**：
   | 级别 | 定义 | 影响 | 目标 |
   |------|------|------|------|
   | P0 - 致命 | 系统崩溃、数据丢失 | 无法使用 | 0 个 |
   | P1 - 严重 | 核心功能不可用 | 严重影响 | ≤1 个 |
   | P2 - 一般 | 功能异常但有替代方案 | 部分影响 | ≤3 个 |
   | P3 - 轻微 | UI 问题、体验问题 | 轻微影响 | ≤5 个 |

3. **质量保障措施**：
   - ✅ 前端单元测试覆盖率 ≥80%
   - ✅ 后端单元测试覆盖率 ≥80%
   - ✅ 集成测试通过率 100%
   - ✅ 联调测试通过率 100%（禁止 Mock）
   - ✅ E2E 测试通过率 100%（禁止 Mock）
   - ✅ 代码审查无 CRITICAL/HIGH 问题
   - ✅ 安全扫描无漏洞
   - ✅ 性能测试达标

4. **测试覆盖要求**：
   - ✅ 正常流程（Happy Path）
   - ✅ 异常流程（Error Handling）
   - ✅ 边界条件（Boundary Cases）
   - ✅ 并发场景（Concurrency）
   - ✅ 数据一致性（Data Consistency）
