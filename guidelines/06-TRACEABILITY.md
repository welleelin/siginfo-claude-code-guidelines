# 可追溯性规范

> 版本：1.0.0
> 最后更新：2026-03-07

---

## 📋 概述

可追溯性确保所有决策和变更都有完整的记录，支持问题排查和知识传承。

---

## 🎯 追溯维度

### 1. 需求追溯

**记录内容**：
- 需求来源
- 需求描述
- 验收标准
- 关联任务

**示例**：
```markdown
## 需求 FR-001: 用户登录

**来源**：产品需求文档 v1.0
**描述**：用户可以使用邮箱和密码登录系统
**验收标准**：
- 支持邮箱密码登录
- 登录失败显示错误提示
- 登录成功跳转到首页

**关联任务**：
- TASK-52: 实现登录功能
- TASK-53: 编写登录测试
```

### 2. 决策追溯

**记录内容**：
- 决策时间
- 决策内容
- 决策原因
- 影响范围

**示例**：
```markdown
## 技术决策 TD-001: 选择 Zustand 作为状态管理

**时间**：2026-03-07
**决策**：使用 Zustand 替代 Redux
**原因**：
- 更轻量（<1KB）
- API 更简洁
- 性能更好

**影响范围**：
- 所有前端状态管理
- 需要重构现有 Redux 代码
```

### 3. 变更追溯

**记录内容**：
- 变更时间
- 变更文件
- 变更原因
- 测试结果

**示例**：
```bash
# Git 提交信息
feat: 实现用户登录功能 (TASK-52)

- 创建登录页面 views/login/index.vue
- 实现认证 API api/auth.ts
- 添加 E2E 测试 e2e/login.spec.ts

需求关联：FR-001
测试覆盖：3 个新测试用例
```

## 📝 记录工具

### MEMORY.md

记录长期记忆：
- 关键决策
- 经验教训
- 用户偏好

### memory/日志

记录每日工作：
- 任务进度
- 技术决策
- 问题解决

### checkpoints/

记录任务检查点：
- 任务状态
- 关键变量
- 测试结果
