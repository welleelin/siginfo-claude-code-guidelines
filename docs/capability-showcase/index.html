专家 Agent 矩阵

9 个专业化 Agent，覆盖软件开发全流程。基于任务复杂度智能分配 Model：Opus(最深推理) → Sonnet(最佳编码) → Haiku(经济高效)

Opus - 复杂架构决策
Sonnet - 主要开发工作
Haiku - 轻量级任务
🎯

planner

Opus

任务规划专家。负责将复杂需求分解为可执行的步骤，识别依赖关系和风险点，创建详细的实施路线图。

核心能力

需求分析 任务分解 依赖识别 风险评估 时间估算

使用场景

  • 收到复杂功能需求时
  • 需要重构大型模块前
  • 多阶段项目需要明确执行顺序

工作流

需求输入 → 理解目标 → 分解子任务 → 建立依赖 → 输出计划
🏗️

architect

Opus

系统架构设计专家。负责技术选型、架构模式选择、可扩展性评估，并记录架构决策 (ADR)。

核心能力

系统架构设计 技术选型 架构模式 可扩展性评估 ADR 记录

使用场景

  • 新项目需要选择技术栈
  • 现有系统需要重构升级
  • 需要在多个技术方案间做决策

工作流

架构问题 → 分析需求 → 对比方案 → 权衡利弊 → 做出决策 → 记录 ADR
✅

tdd-guide

Sonnet

测试驱动开发专家。强制执行 TDD 流程：先写测试 (RED)→实现功能 (GREEN)→重构优化 (IMPROVE)，确保 80%+ 测试覆盖率。

核心能力

TDD 流程 测试用例设计 单元测试 集成测试 覆盖率验证

使用场景

  • 开发新功能时（必须先写测试）
  • 修复 Bug 时（先写复现测试）
  • 重构代码时（确保测试保护）

工作流

RED(写失败测试) → GREEN(实现通过) → IMPROVE(重构优化) → 验证覆盖率
🔍

code-reviewer

Sonnet

代码审查专家。对所有代码变更进行审查，按 CRITICAL/HIGH/MEDIUM/LOW 分级问题，提供具体的修复建议。

核心能力

代码质量审查 问题分级 最佳实践 代码异味检测 性能优化建议

使用场景

  • 写完代码后（必须审查）
  • 提交 PR 前
  • Code Review 会议前预审

工作流

代码提交 → 静态分析 → 问题识别 → 分级标注 → 提供建议
🔒

security-reviewer

Sonnet

安全审查专家。检测安全漏洞和敏感信息，检查 OWASP Top 10 风险，确保代码符合安全最佳实践。

核心能力

漏洞检测 OWASP Top 10 密钥检测 注入攻击防护 认证授权检查

使用场景

  • 实现认证/授权功能后
  • 处理用户输入前
  • 添加 API 端点后

工作流

代码扫描 → 敏感信息检测 → 漏洞识别 → 风险评估 → 修复建议
🔧

build-error-resolver

Sonnet

构建错误解决专家。专门修复构建失败和 TypeScript 类型错误，采用最小改动原则，避免引入新问题。

核心能力

构建错误分析 TypeScript 错误 依赖冲突解决 配置修复 最小改动原则

使用场景

  • npm run build 失败时
  • TypeScript 类型检查报错
  • 依赖安装冲突

工作流

错误分析 → 定位原因 → 生成修复方案 → 最小化修改 → 验证构建
🧹

refactor-cleaner

Sonnet

重构清理专家。使用 knip、depcheck、ts-prune 等工具检测死代码，识别代码重复，安全移除未使用的导入和导出。

核心能力

死代码检测 重复代码识别 导入/导出清理 代码合并 安全删除

使用场景

  • 项目长期未清理技术债务
  • 发现大量未使用的导入
  • 代码重复严重

工作流

运行分析工具 → 识别死代码 → 生成报告 → 确认删除列表 → 执行清理
🎭

e2e-runner

Sonnet

E2E 测试专家。使用 Playwright 创建和执行端到端测试，管理测试旅程，捕获截图/视频/trace，处理 flaky 测试。

核心能力

E2E 测试设计 Playwright Page Object Model 测试旅程管理 截图/视频/Trace

使用场景

  • 需要测试关键用户流程
  • 回归测试自动化
  • 跨浏览器兼容性测试

工作流

识别关键流程 → 设计测试用例 → 编写测试脚本 → 执行测试 → 生成报告
📝

doc-updater

Haiku

文档和 Codemap 维护专家。运行/update-codemaps 和/update-docs 命令，生成 docs/CODEMAPS/*文件，更新 README 和指南。使用经济的 Haiku 模型。

核心能力

Codemap 生成 文档更新 README 维护 API 文档 变更日志

使用场景

  • 代码结构变更后
  • 添加新功能需要文档
  • 准备发布前的文档更新

工作流

分析代码变更 → 更新 Codemap → 同步文档 → 验证链接 → 提交更新

快速开始手册

1

添加为 Git 子模块

将规范库添加到你的项目中

git submodule add https://github.com/your-org/siginfo-claude-code-guidelines.git .guidelines
2

复制模板到项目根目录

初始化项目所需的配置文件

cp -r .guidelines/templates/* ./
3

配置 MCP 服务

在 Claude Code 中配置所需的 MCP 服务

/plugin install bmad-method
/plugin install workflow-studio
4

开始使用命令

使用内置命令开始高效开发

/plan - 创建实施计划 /tdd - TDD 开发流程 /e2e - E2E 测试
