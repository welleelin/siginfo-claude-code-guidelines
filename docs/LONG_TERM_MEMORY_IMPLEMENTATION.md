## 🏗️ 架构设计

### 三层记忆架构

```
┌─────────────────────────────────────────────────────────────┐
│                    OpenClaw 三层记忆架构                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────┐  Hourly 层（短期记忆）                  │
│  │  会话级记忆     │  同步：每小时                          │
│  │  存储：memory/  │  内容：技术决策、实时问题              │
│  └─────────────────┘  检索：实时向量搜索                    │
│                                                             │
│  ┌─────────────────┐  Daily 层（中期记忆）                   │
│  │  每日归档       │  同步：每日 23:00                       │
│  │  存储：memory/  │  内容：项目进展、重要决策              │
│  └─────────────────┘  检索：时间线 + 标签                    │
│                                                             │
│  ┌─────────────────┐  Weekly 层（长期记忆）                  │
│  │  周度总结       │  同步：每周日 22:00                     │
│  │  存储：MEMORY.md│  内容：核心知识、最佳实践              │
│  └─────────────────┘  检索：语义搜索 + 关联分析             │
│                                                             │
│  ┌─────────────────┐  Context 层（工作记忆）                 │
│  │  当前会话       │  同步：实时                            │
│  │  存储：Context  │  内容：对话历史、工具调用              │
│  └─────────────────┘  管理：70%/80%/90% 阈值                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 同步机制

| 层级 | 频率 | 时间 | 脚本 |
|------|------|------|------|
| Hourly | 每小时 | :00 | `sync-hourly.sh` |
| Daily | 每日 | 23:00 | `archive-daily.sh` |
| Weekly | 每周 | 周日 22:00 | `summarize-weekly.sh` |

### 检查点类型

| 类型 | 触发条件 | 脚本 |
|------|---------|------|
| 任务开始 | 开始新任务 | `checkpoint.sh start` |
| 关键节点 | 完成重要步骤 | `save-state.sh` |
| 小时同步 | 每小时 | `sync-hourly.sh` (自动) |
| 日终归档 | 每日 23:00 | `archive-daily.sh` (自动) |
| 任务完成 | 完成任务 | `checkpoint.sh complete` |

## 🔧 核心功能

### 1. 记忆文件管理

- 自动创建目录结构
- 模板文件复制
- 今日日志创建
- Git 忽略配置

### 2. 状态保存与恢复

**保存内容**：
- 任务信息（ID、阶段、步骤）
- Git 状态（分支、Commit、更改）
- 文件列表（已修改、未跟踪）
- 记忆文件状态

**恢复流程**：
1. 查找检查点文件
2. 解析检查点数据
3. 验证 Git 状态
4. 生成恢复指南
5. 提供下一步建议

### 3. 同步机制

**Hourly 同步**：
- 记录当前任务状态
- 更新今日日志 Hourly 层
- 更新心跳状态

**Daily 归档**：
- 生成日报摘要
- 更新 MEMORY.md
- 准备明日待办
- 生成归档报告

**Weekly 总结**：
- 生成周报（聚合本周日志）
- 更新 MEMORY.md（长期记忆层）
- 清理旧日志（可选）
- 生成周报摘要

### 4. 记忆检索

**支持的查询类型**：
- 精确查询（关键词匹配）
- 语义搜索（向量检索）
- 时间范围（历史回溯）
- 标签过滤（多维分析）

**检索优先级**：
1. MEMORY.md（长期记忆）
2. memory/今日.md + 昨日.md（短期记忆）
3. memory/历史日志（中期记忆）
4. checkpoints/*.json（状态检查点）

## 🚀 使用方法

### 初始化

```bash
# 在新项目中初始化记忆系统
cd /path/to/your/project
/path/to/sig-claude-code-guidelines/scripts/init-memory.sh
```

### 日常使用

```bash
# 1. 开始任务
./scripts/checkpoint.sh start 52

# 2. 保存状态（关键节点）
./scripts/save-state.sh "完成需求分析"

# 3. 小时同步（可自动）
./scripts/sync-hourly.sh

# 4. 日终归档（可自动）
./scripts/archive-daily.sh

# 5. 恢复状态（需要时）
./scripts/restore-state.sh latest
```

### 配置 Cron（自动同步）

```bash
# 编辑 crontab
crontab -e

# 添加以下条目
0 * * * * /path/to/sync-hourly.sh
0 23 * * * /path/to/archive-daily.sh
0 22 * * 0 /path/to/summarize-weekly.sh
```

## 🔒 安全与隐私

### 敏感信息处理

**禁止写入记忆文件**：
- API Key、密码、Token
- 服务器 IP 地址（使用占位符）
- 个人联系方式
- 数据库连接字符串

**正确做法**：
- 敏感信息存入 `~/.openclaw/credentials/`
- 或使用环境变量

### Git 备份配置

```gitignore
# .gitignore 推荐配置
.DS_Store
.key
.pem
.secrets/
~/.openclaw/credentials/
*.local
checkpoints/*.json
.memory-archive/
```

### MEMORY.md 加载规则

| 会话类型 | 是否加载 MEMORY.md | 说明 |
|---------|------------------|------|
| 主会话 (1 对 1) | ✅ 加载 | 私密环境 |
| 群组对话 | ❌ 不加载 | 公共场合，避免泄露隐私 |
