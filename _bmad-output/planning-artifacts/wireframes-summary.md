---

## 📐 线框图详情

### 1. 主页面布局（Notion 风格）

**尺寸**: 1440x900px
**布局**: 左侧导航 (280px) + 内容区域 (居中 800px)

**关键元素**:
- **左侧导航区**:
  - 搜索框 (Cmd+K 快捷键提示)
  - 文档树形导航
  - 当前选中状态高亮 (#e8e7e3)

- **内容区域**:
  - 面包屑导航
  - 大标题 (36px, 粗体)
  - 副标题 (16px, 行高 1.7)
  - 内容最大宽度 800px，居中显示

**设计特点**:
- 温暖背景色 (#f7f6f3) 用于导航区
- 纯白背景 (#ffffff) 用于内容区
- 充足的留白和宽松的行高
- 符合 Notion 风格的视觉层次

---

### 2. 搜索交互状态

**尺寸**: 1440x900px
**交互状态**: 搜索激活 + 结果展示

**关键元素**:
- **激活的搜索框**:
  - 蓝色边框 (#0066CC, 2px) 表示焦点状态
  - 实时输入 "TDD"

- **搜索结果列表**:
  - 结果数量提示 "找到 3 个结果"
  - 第一个结果高亮 (#e8e7e3)
  - 每个结果显示标题 + 文件路径

- **内容预览**:
  - 匹配内容高亮显示
  - 黄色高亮框 (#FFF9E6, 边框 #FFD700)

**交互流程**:
1. 用户输入关键词 "TDD"
2. 实时显示 3 个匹配结果
3. 第一个结果自动选中
4. 右侧预览匹配内容

---

### 3. 学习路径页面

**尺寸**: 1440x900px
**用途**: 新用户引导和进度追踪

**关键元素**:
- **左侧导航**:
  - 学习进度条 (3/14 已完成)
  - 进度百分比可视化

- **主内容区**:
  - 页面标题 "🎓 新手学习路径"
  - 3 个学习卡片:
    1. **系统总则** - ✅ 已完成 (蓝色边框)
    2. **行动准则** - 🔄 进行中 (蓝色文字)
    3. **TDD 工作流程** - ⏳ 待学习 (灰色)

- **成就徽章**:
  - 🏆 "快速入门" 徽章
  - 浅蓝色背景 (#F0F9FF)
  - 鼓励用户继续学习

**设计特点**:
- 清晰的状态指示（已完成/进行中/待学习）
- 预计学习时间提示
- 游戏化元素（徽章、进度条）

---

### 4. 移动端响应式布局

**尺寸**: 375x812px (iPhone X/11/12 标准尺寸)
**布局**: 顶部导航 + 内容区 + 抽屉导航

**主屏幕**:
- **顶部导航栏** (56px):
  - 左侧：汉堡菜单按钮
  - 中间：页面标题 "Guidelines"
  - 右侧：搜索按钮

- **内容区域**:
  - 面包屑导航 (12px)
  - 标题 (28px, 适配移动端)
  - 正文内容 (14px, 行高 1.7)
  - 左右内边距 20px

**抽屉导航** (280x812px):
- 顶部：标题 + 关闭按钮
- 搜索框
- 文档列表（当前选中高亮）
- 学习进度条
- 完成数量统计

**响应式策略**:
- 导航折叠为抽屉
- 内容全宽显示
- 触摸友好的按钮尺寸 (44x44px)
- 简化的视觉层次

## 🎯 设计原则应用

### 1. Notion 风格一致性

| 元素 | 设计参数 |
|------|---------|
| 导航背景 | #f7f6f3 (温暖色调) |
| 内容背景 | #ffffff (纯白) |
| 内容最大宽度 | 800px |
| 标题字号 | 36px (桌面) / 28px (移动) |
| 正文字号 | 16px (桌面) / 14px (移动) |
| 行高 | 1.7 |
| 圆角 | 8px |

### 2. 用户旅程支持

| 旅程 | 线框图支持 |
|------|-----------|
| **快速查找规范** | 搜索交互状态 - 展示实时搜索和结果高亮 |
| **学习新规范** | 学习路径页面 - 展示进度追踪和引导流程 |
| **深度探索** | 主页面布局 - 展示完整的导航和内容结构 |
| **移动端访问** | 移动端布局 - 展示响应式适配策略 |

### 3. 无障碍设计

| 要求 | 实现 |
|------|------|
| 触摸目标尺寸 | 所有按钮 ≥ 40x40px (移动端 44x44px) |
| 颜色对比度 | 文字与背景对比度 > 4.5:1 |
| 焦点指示 | 搜索框激活时显示蓝色边框 (2px) |
| 状态反馈 | 清晰的视觉状态（已完成/进行中/待学习）|

## 📝 设计决策记录

### 为什么选择 Notion 风格？

1. **阅读体验优先**: 内容居中 800px，减少眼球移动
2. **温暖舒适**: #f7f6f3 背景色降低视觉疲劳
3. **现代简洁**: 符合当前设计趋势，体现专业性
4. **差异化定位**: 不同于传统的 GitHub 文档风格

### 为什么创建学习路径页面？

1. **降低学习曲线**: 新用户需要明确的学习路径
2. **进度可视化**: 进度条和徽章提供即时反馈
3. **游戏化激励**: 成就系统鼓励用户完成学习
4. **符合用户旅程**: 支持 "学习新规范" 核心旅程

### 为什么优先设计移动端？

1. **移动优先策略**: 确保核心功能在小屏幕可用
2. **响应式验证**: 测试布局在极端尺寸下的表现
3. **触摸交互**: 验证按钮尺寸和交互区域
4. **真实使用场景**: 开发者可能在移动设备上查阅规范
