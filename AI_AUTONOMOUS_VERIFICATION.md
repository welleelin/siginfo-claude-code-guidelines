# AI 自主验证规范

> **版本**：1.0.0
> **最后更新**：2026-03-12
> **核心变更**：AI 必须自动验证预期效果，不能等用户发现问题

---

## 📋 概述

### 问题背景

**❌ 错误做法**：
```
AI 完成开发后：
"功能已开发完成，请查看效果"

用户访问后发现问题：
- 页面空白
- 工具栏不显示
- 控制台报错
- 功能不可用
```

**后果**：用户成为测试者，体验极差。

### 正确做法

**✅ AI 自主验证流程**：
```
AI 完成开发后，必须自动执行：

1. 启动验证 - 确认服务已启动，端口可访问
2. 视觉验证 - 访问页面，检查预期 UI 元素是否显示
3. 功能验证 - 点击交互元素，验证功能是否正常
4. 日志验证 - 检查控制台是否有报错
5. 截图验证 - 截图并审查是否有视觉问题

只有所有验证通过，AI 才能告知用户：
"功能已完成，可直接访问 http://localhost:3001"
```

## 🎯 预期效果验证案例

### 案例：Agentation 工具栏集成

#### 预期效果描述

在浏览器中访问 `http://localhost:3001` 应该显示：

1. **游戏列表页面**

   - 选项卡（Tab 导航）

2. **Agentation 标注工具**

## 🔧 验证脚本

### 使用方式

```bash
# 运行预期效果验证
npx tsx scripts/verify-expectations.ts http://localhost:3001

# 输出示例：
🔍 开始验证：http://localhost:3001
============================================================
✅ 页面加载成功
✅ 统计卡片已显示 (3 个)
✅ 选项卡已显示
✅ 游戏网格已显示 (12 个游戏)
✅ Agentation 工具栏已显示
✅ 工具栏位置正确（右下角）
✅ 工具栏颜色正常：rgb(186, 104, 200)
✅ 控制台无错误
✅ 截图已保存：verification-result.png

============================================================
📊 验证结果
============================================================
✅ 所有验证通过！功能已完成，可直接使用。

访问地址：http://localhost:3001
```

### 脚本功能

| 功能 | 说明 |
|------|------|
| **服务可访问性** | 验证 HTTP 状态码 = 200 |
| **游戏列表验证** | 统计卡片、选项卡、游戏网格 |
| **工具栏验证** | 存在性、位置、颜色、可交互 |
| **控制台检测** | 收集 JavaScript 错误 |
| **网络请求检测** | 检测 4xx/5xx 失败请求 |
| **截图审查** | 保存完整页面截图 |
| **Bug 上报** | 发现问题自动创建 Bug 报告 |

## 📊 验证脚本配置

### 可配置选项

```typescript
verifyExpectations(url, {
  headless: true,           // 是否无头模式
  timeout: 30000,           // 超时时间（毫秒）
  verifyGameList: true,     // 是否验证游戏列表
  verifyAgentation: true,   // 是否验证工具栏
})
```

### 自定义验证

```typescript
import { verifyExpectations } from './scripts/verify-expectations'

// 自定义验证逻辑
const result = await verifyExpectations('http://localhost:3001', {
  headless: false,  // 显示浏览器（调试用）
  verifyGameList: false,  // 跳过游戏列表验证
  verifyAgentation: true,  // 只验证工具栏
})

if (result.passed) {
  console.log('✅ 验证通过')
} else {
  console.log('❌ 发现问题:', result.errors)
}
```

## 📋 检查清单

### AI 自主验证专项检查

在 AI 告知用户"功能已完成"之前，必须完成以下验证：

- [ ] **服务可访问性**：目标端口可访问，HTTP 返回 200
- [ ] **页面加载验证**：页面在 3 秒内完成加载
- [ ] **预期 UI 元素**：所有预期 UI 元素可见
- [ ] **视觉完整性**：无 404/空白/错位
- [ ] **控制台无错误**：无 JS 错误、无 404 请求
- [ ] **功能可交互**：关键按钮/链接可点击并响应
- [ ] **截图审查通过**：截图验证无视觉问题
- [ ] **Bug 自动上报**：发现问题自动创建 Bug 报告
