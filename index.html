📋 Phase 2 详解：三轨道任务规划

根据任务复杂度自动匹配规划轨道，避免小任务过度设计、大任务欠缺规划。

Quick Flow
快速开发
⏱ 适合 < 2 小时的任务
  • Bug 修复 / 小功能
  • /plan "任务描述"
  • 可选 Quick Spec
  • 用户确认后直接开发
  • bmm-quick-flow-solo-dev
Standard
标准开发
⏱ 适合 2–8 小时的任务
  • 新功能 / 模块重构
  • /bmad-help 智能诊断

  • BMAD 完整流程
  • 多 Agent 并行协作
  • ADR 架构决策记录
  • 多轮 Code Review

⚡ 行动准则

开发时怎么干才不会出问题？照着这个流程走就对了。

🎯 核心原则（7 条铁律）

1️⃣
先想再做

写代码前必须规划，想清楚要做什么再动手

2️⃣
质量优先

代码质量过关才能测试，测试过关才能提交

3️⃣
测试驱动

先写测试用例（让它失败），再写代码（让它通过）

4️⃣
API 完整

所有 API 都开发完了才能联调测试

5️⃣
安全第一

安全检查通过才能发布，不能有漏洞上线

6️⃣
文档同步

改了代码就要更新文档，别让人看不懂

7️⃣
能追溯

所有决策和变更都要记下来，出了问题能找到原因

🔄 综合性测试工作流 v2.0

软件工程"一头一尾"原则：需求阶段是"头"，测试阶段是"尾"，两头抓牢才能开发顺畅

0️⃣
Phase 0
需求文档验证
1️⃣
Phase 1
前端 UI 测试
2️⃣
Phase 2
后端 API 测试
3️⃣
Phase 3
前后端联调

📌 Phase 0: 需求文档验证（测试前置）

✓ 项目类型确认：新项目还是旧项目二次开发？
✓ 需求收集迭代：用户描述 → AI 整合 → 确认 → 补充 → 闭环
✓ 界面原型确认：ASCII/命令行初步展示，让用户确认布局
✓ 验收清单生成：作为最终验收的唯一依据
💡 核心原则：需求不清晰，测试就没有依据。必须先验证需求文档，再进入开发测试。

📌 Phase 3: 联调测试 6 个子阶段

3.1 环境准备：服务启动、网络检查、Mock 清理
3.2 数据准备：清理脏数据、填充基础数据、验证依赖
3.3 初步联调：功能正确性、控制台无 Bug
3.4 业务流程：跨功能流程、状态流转、权限边界
3.5 多角色协作：角色权限、数据隔离、审批流程
3.6 Bug 记录：人性化描述、解决方案、验证结果

🎭 Mock 使用规则（什么时候能用假数据）

测试阶段 能用 Mock？ 说明
Phase 0: 需求验证 — 不适用 需求阶段不需要数据，只需要文档验证
Phase 1: 前端 UI 测试 ✅ 允许 前端 Mock 模式，验证 UI 和交互
Phase 2: 后端 API 测试 ❌ 禁止 必须用真实数据库和真实数据
Phase 3: 前后端联调 ❌ 严禁 必须用完整真实环境

💡 简单记：只有 Phase 1 前端测试可以用 Mock，其他阶段必须用真实数据

⚠️ 前后端联调测试的铁律

1️⃣
必须用真实 API，禁止 Mock

联调就是测真实对接，用 Mock 等于没测

2️⃣
数据准备阶段不能跳过

清理脏数据 → 填充基础数据 → 验证依赖关系

3️⃣
业务流程测试 ≠ 单体功能测试

测的是完整流程：浏览 → 加购 → 结算 → 支付 → 完成

4️⃣
多角色协作测试必须做

模拟买家、卖家、管理员不同角色的操作流程

5️⃣
Bug 描述用人话

什么情况下 + 做了什么 + 期望什么 + 实际发生了什么

# 联调测试检查清单
□ 数据准备：清理脏数据、填充基础数据
□ 环境验证：前端连接真实后端 API
□ 功能联调：所有功能正常使用
□ 业务流程：完整购买流程通过
□ 多角色：买家/卖家/管理员协作正常
□ 控制台：无红色 Error
□ 网络：无 4xx/5xx 错误

使用多个 Agent 并行执行测试，提高效率和覆盖率

⚡

✅ 依赖清晰

  • Phase 3.5 多角色 → Swarm 模式（买家/卖家/管理员交叉验证）

🏪 真实业务全流程闭环测试（REAL Testing）

传统测试只停留在 UI 元素存在层面，用 Mock 数据，无法验证真实业务闭环。REAL Testing 要求四阶段逐步升级。

1️⃣
Phase 1
前端 Mock 测试
✅ 允许 Mock
2️⃣
Phase 2
API 阶段测试
❌ 禁止 Mock
3️⃣
Phase 3
前后端联调
❌ 严禁 Mock
4️⃣
Phase 4
真实业务闭环
真实用户 + 数据
Phase 1 - 前端 Mock 测试： 验证所有按钮/样式、前端流程点击闭环、所有页面展现。产出：前端测试报告 + 未执行测试清单
Phase 2 - API 测试： 多 Agent 并行执行，覆盖所有功能点和用户故事。产出：API 测试报告（禁止 Mock）
Phase 3 - 联调测试： 完整业务流程（下单→支付→完成），多角色协作（买家/卖家/管理员）。真实环境，严禁 Mock
Phase 4 - 闭环验证： 真实用户操作 + 真实数据验证，生成可用性评估报告，确认生产级质量
📄 详见 guidelines/12-REAL_BUSINESS_TESTING.md

🛡️ Shannon AI 渗透测试

Shannon 是 Keygraph 开发的自主 AI 渗透测试工具，通过分析源代码、执行真实攻击来发现可利用的安全漏洞。

🔍
白盒测试
源代码分析识别攻击面，API 接口自动发现，代码路径追踪漏洞点
🎯
真实攻击验证
OWASP Top 10、SQL 注入、XSS、SSRF、认证/授权漏洞，只报告可利用的漏洞
📋
PoC 报告
提供可复现的漏洞证明，包含修复建议和严重程度评级
🔗
Claude Code 集成
通过 skill 自动触发，无缝集成到开发流程的 Step 7
使用方法：
/shannon-start <URL> 启动安全测试（自动分析代码 + 攻击）
/shannon-logs 查看测试进度和发现的漏洞
/shannon-report 生成完整安全测试报告

🎯 确定性开发规范

消除 AI 开发中的不确定性，让每次执行结果可预期、可重复、可验证。

📌

🔄

  • 有需求变更文档
  • 用户明确确认
  • 回归测试通过

plan 需要规划任务、评估风险时 明确需求 → 评估风险 → 生成步骤计划，等用户确认再执行
bmad-bmm-* 说"创建 PRD"、"创建架构"、"规划 Sprint"时 BMAD Method 全套规划工具：PRD、架构设计、Epic、Story、Sprint 规划
💻 开发类 Skills
Skill 名称 什么时候自动激活 能干什么
coding-standards 写代码、审查代码、重构时 帮你写出规范的代码，命名、格式、结构都到位
tdd-workflow 写新功能、修 Bug、重构时 强制先写测试再写代码，保证 80% 测试覆盖率
e2e-testing 写 E2E 测试、Playwright 相关时 自动生成端到端测试，模拟真实用户操作
api-design 设计 API、写接口文档时 帮你设计规范的 REST API，URL、参数、错误码都有规范
backend-patterns 写后端代码、Node.js/Express 时 后端最佳实践：架构、API、数据库优化
frontend-patterns 写前端代码、React/Next.js 时 前端最佳实践：组件、状态管理、性能优化
postgres-patterns 写 SQL、设计数据库时 PostgreSQL 最佳实践：索引、查询优化、安全配置
🔒 安全类 Skills
Skill 名称 什么时候自动激活 能干什么
agentation 说"添加视觉反馈工具"、"安装 Agentation"时 为 Next.js 项目添加 Agentation 视觉反馈工具栏，支持 AI 标注和设计评审
agentation-self-driving 说"自动标注"、"批评这个页面"、"自驾模式"时 自主设计批评模式：AI 自动在页面上添加设计反馈注释，无需手动操作
shannon 说"安全测试"、"渗透测试"、"Shannon"时 运行 Shannon AI 渗透测试，自动发现并验证安全漏洞
security-review 写认证、处理用户输入、用密钥时 全面安全检查：认证、注入、XSS、CSRF、密钥管理
security-scan 修改配置文件、提交前检查时 扫描 .claude/ 目录的安全漏洞，防注入、防泄密
✅ 验证类 Skills
Skill 名称 什么时候自动激活 能干什么
verification-loop 完成功能、创建 PR、重构后 全面验证：构建、类型检查、测试、代码审查
eval-harness 需要评估 AI 工作质量时 评估框架，量化 AI 输出质量
simplify 代码有改动时 检查代码是否可以复用、是否有冗余、是否高效
💰 成本优化类 Skills
Skill 名称 什么时候自动激活 能干什么
cost-aware-llm-pipeline 调用 LLM API、处理批量任务时 自动选便宜的模型、追踪花费、缓存 Prompt，帮你省钱
🤖 AI 辅助类 Skills
Skill 名称 什么时候自动激活 能干什么
iterative-retrieval 启动子 Agent、需要代码上下文时 渐进式搜索信息，解决"AI 不知道要什么"的问题
continuous-learning-v2 AI 完成任务、发现新模式时 自动学习新技能，把经验变成可复用的技能包
📄 文档处理类 Skills
Skill 名称 什么时候自动激活 能干什么
nutrient-document-processing 处理 PDF、Word、Excel 时 文档转换、OCR 识别、提取内容、添加水印
🌉 桥接类 Skills
Skill 名称 什么时候自动激活 能干什么
claude-to-im 说"桥接"、"消息推送"、"连飞书"时 把 Claude 连到微信/Telegram/飞书/钉钉/短信，手机上也能用（基于 cc-connect）
claude-api 代码里有 anthropic、@anthropic-ai/sdk 时 帮你用 Claude API 开发应用

🔄 是否检查已安装工具的更新？

检查更新 跳过 以后不再询问
选项 说明
检查更新 立即检查所有已安装插件和 Skills 是否有更新
跳过 本次启动不检查，下次启动仍会询问
以后不再询问 记录配置，以后启动不再询问（可手动恢复）

/check-updates --enable-ask 恢复启动时询问
/check-updates --disable-ask 禁用启动时询问
